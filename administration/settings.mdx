## General Settings

| Setting | Description |
|---------|-------------|
| **Platform name** | Your company/platform name |
| **Primary domain** | The main domain for the panel (e.g. `panel.yourcompany.com`) |
| **Default language** | UI language for new users |
| **Timezone** | Server and UI timezone (affects cron and schedules) |
| **Date format** | How dates are displayed in the UI |
| **Registration** | Allow/disallow public user registration |
| **Platform mode** | SaaS or WaaS |

## White Label

Customize the platform appearance for your clients:

| Setting | Description |
|---------|-------------|
| **Brand name** | Replaces "AreaWP" throughout the UI |
| **Logo (light mode)** | SVG or PNG, recommended 200×50px |
| **Logo (dark mode)** | Logo variant for dark theme |
| **Favicon** | Browser tab icon (32×32 ICO or PNG) |
| **Mode** | `Text only` (replace text only) or `Full logo` (replace logo + text) |
| **Primary color** | Main accent color (hex) |
| **Support URL** | Link to your support portal |
| **Hide version** | Remove version numbers from the footer |

### White Label Mode

<Tabs>
  <Tab title="Text Only">
    AreaWP logos are kept, but all text references to "AreaWP" are replaced with your brand name. Easiest to set up — just enter your brand name and save.
  </Tab>
  <Tab title="Full Logo">
    Both the logos and all text references are replaced. Requires uploading your own logo files. Fully white-labeled experience.
  </Tab>
</Tabs>

## Security Settings

| Setting | Default | Description |
|---------|---------|-------------|
| **Enforce 2FA for admins** | Off | Require 2FA for all admin accounts |
| **Session timeout** | 120 min | Auto-logout after inactivity |
| **Max login attempts** | 5 | Fail2ban trigger threshold |
| **Password policy** | Medium | Minimum password strength |
| **API rate limit** | 100 req/min | Requests per minute per IP |
| **Allowed IP ranges** | Any | Restrict admin panel access by IP |

## Email / SMTP

Configure the outgoing email server for platform notifications, invoices, and user communications:

| Field | Description |
|-------|-------------|
| **SMTP host** | Mail server hostname (e.g. `smtp.sendgrid.net`) |
| **SMTP port** | 587 (STARTTLS) or 465 (SMTPS) |
| **Username** | SMTP authentication username |
| **Password** | SMTP authentication password |
| **From email** | Sender address (e.g. `noreply@yourplatform.com`) |
| **From name** | Sender display name |

### Testing SMTP

Click **Send Test Email** after saving SMTP settings. AreaWP sends a test message to the admin email address to verify the configuration.

### Email Templates

Customize the content and branding of platform emails:

| Template | Trigger |
|---------|---------|
| Welcome email | New user registration |
| Email verification | Account created, requires verification |
| Password reset | User requests password reset |
| Invoice | Payment processed |
| Subscription renewal | Upcoming renewal reminder |
| Payment failed | Stripe charge failed |
| Subscription canceled | Subscription canceled |
| Backup complete | Backup job finished |
| Backup failed | Backup job error |

## License

| Field | Description |
|-------|-------------|
| **License key** | Your AreaWP license key (`xxxxx-xxxxx-xxxxx`) |
| **Status** | Active / Suspended / Expired |
| **Domain** | Domain the license is registered to |
| **Plan** | License tier (Starter / Pro / Business / Enterprise) |
| **Max sites** | Site limit per your license |
| **Expires** | License expiry date |

### Validating / Refreshing

Click **Validate License** to force a fresh check against the AreaWP license server. Use this after purchasing, renewing, or changing your domain.

<Info>
License validation happens automatically every 6 hours. If the server is offline, AreaWP uses a cached result valid for up to 14 days.
</Info>

## Notifications

Configure where platform alerts are sent:

| Channel | Usage |
|---------|-------|

Configure notification rules by category: billing, security, backups, uptime, system.

## Sidebar Customization

Adjust the default sidebar layout for all users:

1. Go to **Settings → Sidebar**
2. Choose the default preset (Full, Web Hosting, DevOps, etc.)
3. Save — new users start with this preset

Users can still customize their own sidebar from the user menu.
