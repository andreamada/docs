## Stripe

Stripe enables fully automated credit and debit card payments with automatic subscription renewals.

### Setting Up Stripe

<Steps>
  <Step title="Create a Stripe account">
    Sign up at [stripe.com](https://stripe.com) and complete account verification.
  </Step>

  <Step title="Get your API keys">
    In the Stripe dashboard, go to **Developers → API Keys** and copy your:
    - **Publishable key** (`pk_live_...`)
    - **Secret key** (`sk_live_...`)

    Use **test keys** (`pk_test_...` / `sk_test_...`) during development.
  </Step>

  <Step title="Enter keys in AreaWP">
    Go to **Payment Gateways → Stripe** and paste both keys. Click **Save**.
  </Step>

  <Step title="Configure webhook (recommended)">
    For reliable event processing (payment failures, disputes, refunds), configure a Stripe webhook:

    1. In AreaWP, go to **Payment Gateways → Stripe → Webhook**
    2. Copy the webhook endpoint URL
    3. In Stripe dashboard → **Developers → Webhooks** → **Add Endpoint**
    4. Paste the URL and select events:
       - `payment_intent.succeeded`
       - `payment_intent.payment_failed`
       - `customer.subscription.deleted`
       - `invoice.payment_succeeded`
       - `invoice.payment_failed`
    5. Copy the **Signing secret** and paste it back in AreaWP
  </Step>

  <Step title="Test the integration">
    Go to **Payment Gateways → Stripe → Test**. AreaWP attempts a $0.01 authorization to verify the connection.
  </Step>
</Steps>

### Stripe Configuration Options

| Setting | Description |
|---------|-------------|
| **Currency** | Default billing currency (e.g. USD, EUR, GBP) |
| **Statement descriptor** | What appears on client's bank statement |
| **Trial cancellation** | Cancel trial if no card added by end date |
| **Failed payment retry** | Retry schedule for failed payments |
| **3D Secure** | Require 3DS verification for applicable cards |

## Bank Transfer

Bank Transfer enables manual payments where clients send money via bank wire and admins confirm receipt.

### Setting Up Bank Transfer

1. Go to **Payment Gateways → Bank Transfer**
2. Enable the gateway
3. Enter your bank account instructions:

```
Please transfer the amount to:
Bank: First National Bank
Account name: Your Company Ltd
IBAN: GB00 XXXX 0000 0000 0000 00
BIC/SWIFT: XXXXXXXX
Reference: [Your invoice number]

Payments are processed within 1-3 business days.
```

### Bank Transfer Workflow

1. Client selects **Bank Transfer** at checkout
2. AreaWP creates a **Pending** invoice and shows the bank details
3. Client sends the bank transfer
4. Admin receives the payment (external to AreaWP)
5. Admin goes to **Transactions** and marks the invoice as **Paid**
6. Subscription activates automatically

<Tip>
Ask clients to include their invoice number as the payment reference to make reconciliation easier.
</Tip>

## Testing Payments

Before going live, test your payment setup:

### Stripe Test Cards

| Card Number | Behavior |
|------------|---------|
| `4242 4242 4242 4242` | Always succeeds |
| `4000 0000 0000 0002` | Always declined |
| `4000 0025 0000 3155` | Requires 3D Secure |
| `4000 0000 0000 9995` | Insufficient funds |

Use any future expiry date, any CVC, and any ZIP code.

## Currency Configuration

Set your platform's currency in **Settings → Billing → Currency**. The currency applies to all pricing plans and invoices.

<Warning>
Changing the currency after clients have active subscriptions is not recommended — existing invoices will show the old currency. Make this decision before going live.
</Warning>

## Tax / VAT

Configure tax collection in **Settings → Billing → Tax**:

| Setting | Description |
|---------|-------------|
| **Tax rate** | Percentage to add (e.g. 20% VAT) |
| **Tax label** | Label on invoice (VAT, GST, Sales Tax) |
| **Tax number** | Your business tax registration number |
| **Show tax** | Display tax separately on invoices |
| **Stripe Tax** | Use Stripe's automatic tax calculation (Stripe only) |
