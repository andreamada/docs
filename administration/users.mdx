## User Roles

AreaWP has two primary roles:

| Role | Access Level |
|------|-------------|
| **Admin** | Full platform access — all settings, all servers, all client data |
| **User** | Client access — only their own sites, billing, and support |

### Role Permissions

| Feature | Admin | User |
|---------|-------|------|
| Server management | ✅ Full | ❌ None |
| Docker, databases, files | ✅ Full | ❌ None |
| Site management (all sites) | ✅ Full | Own sites only |
| Billing and subscriptions | ✅ Full | Own billing only |
| User management | ✅ Full | ❌ None |
| Platform settings | ✅ Full | ❌ None |
| Activity log | ✅ Full | ❌ None |

## User List

The Users page shows all accounts with:

| Column | Description |
|--------|-------------|
| **Name** | Full name |
| **Email** | Login email address |
| **Role** | Admin or User |
| **Status** | Active / Suspended / Pending |
| **Subscription** | Current plan (if applicable) |
| **Last Login** | Most recent login timestamp |
| **Joined** | Account creation date |

## Creating a User

### Admin-Created Account

1. Go to **Users** → **New User**
2. Fill in name, email, and role
3. Set a password (or send a password reset email)
4. Optionally assign a subscription/plan
5. Click **Create**

### Invitation Flow

Send an email invitation for self-registration:

1. Click **Invite User**
2. Enter the email address
3. Set the role they'll have after registration
4. Optionally pre-assign a pricing plan
5. Click **Send Invitation**

The user receives an email with a registration link valid for 48 hours.

### Registration (Self-Service)

If registration is enabled in **Settings → Registration**, users can sign up themselves at `/register`.

## User Actions

| Action | Description |
|--------|-------------|
| **Suspend** | Block login while preserving account and data |
| **Activate** | Re-enable a suspended account |
| **Impersonate** | Log in as the user to troubleshoot (admin logs show the impersonation) |
| **Reset Password** | Send a password reset email |
| **Delete** | Permanently delete account (requires confirmation) |
| **Change Role** | Promote to admin or demote to user |
| **Assign Plan** | Set the user's pricing plan manually |

---

## Account Suspension

Suspending a user:
- Prevents login to the platform
- Does not delete any data or sites
- Sends an optional notification email
- Can be reversed at any time

Reasons for suspension:
- Non-payment (automated by billing system)
- Terms of service violation (manual)
- Fraudulent activity

## Two-Factor Authentication (2FA)

Admins can view each user's 2FA status and reset it if they lose access to their authenticator app.

1. Go to the user detail page → **Security**
2. Click **Reset 2FA** to disable and require re-enrollment on next login

<Tip>
Require 2FA for all admin accounts. Enable this in **Settings → Security → Enforce 2FA for Admins**.
</Tip>
