**Path:** Email Server (`/email`) — Admin only

The Email Server section manages Postfix (SMTP) and Dovecot (IMAP/POP3) on your connected servers, including domain configuration, mailboxes, and deliverability.
</Info>

## Overview

AreaWP can configure a full mail server stack on any connected Linux server:

| Component | Software | Protocol |
|-----------|---------|---------|
| **SMTP (sending)** | Postfix | Port 25, 465 (SMTPS), 587 (submission) |
| **IMAP (receiving)** | Dovecot | Port 143, 993 (IMAPS) |
| **POP3 (receiving)** | Dovecot | Port 110, 995 (POP3S) |
| **Spam filter** | SpamAssassin / Rspamd | (integrated) |
| **Webmail** | Roundcube | HTTPS |

## Setting Up a Mail Domain

<Steps>
  <Step title="Add the domain">
    Go to **Email Server → Domains** → **Add Domain**. Enter the domain you want to host email for (e.g. `example.com`).
  </Step>

  <Step title="Configure DNS records">
    AreaWP shows the required DNS records. Add them to your DNS provider:

    | Record | Type | Value |
    |--------|------|-------|
    | `example.com` | MX | `mail.example.com` (priority 10) |
    | `mail.example.com` | A | Your server IP |
    | `example.com` | TXT | `v=spf1 ip4:YOUR_IP ~all` |
    | `_dmarc.example.com` | TXT | `v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com` |
    | `default._domainkey.example.com` | TXT | DKIM public key (generated by AreaWP) |
  </Step>

  <Step title="Generate DKIM key">
    Click **Generate DKIM** to create a DKIM keypair for the domain. AreaWP shows the DNS TXT record to add for DKIM verification.
  </Step>

  <Step title="Provision SSL">
    Enable SSL for IMAP/SMTP using Let's Encrypt. AreaWP provisions the certificate for `mail.example.com` automatically.
  </Step>
</Steps>

## Mailboxes

### Creating a Mailbox

1. Go to **Email Server → Mailboxes** → **New Mailbox**
2. Configure:

| Field | Description |
|-------|-------------|
| **Email address** | `user@example.com` |
| **Password** | Mailbox password (stored hashed) |
| **Quota** | Storage limit (e.g. `2GB`) |
| **Domain** | Which configured email domain |

### Mailbox Operations

| Operation | Description |
|-----------|-------------|
| **Reset password** | Set a new mailbox password |
| **Quota usage** | View current storage used |
| **Enable/disable** | Temporarily disable without deleting |
| **Webmail login** | Open Roundcube with auto-login |
| **Delete** | Permanently remove mailbox and all messages |

## SMTP Relay Configuration

If you prefer to use an external SMTP provider for sending (SendGrid, Amazon SES, Mailgun, etc.) instead of a local Postfix server:

1. Go to **Email Server → SMTP Relay**
2. Enter the relay server credentials
3. AreaWP configures Postfix to use the relay for all outgoing mail

This is useful for avoiding spam blocklist issues on new IP addresses.

## Deliverability

### Checking Deliverability

AreaWP includes a built-in deliverability check:

1. Go to **Email Server → Deliverability**
2. Click **Run Check**

The check verifies:
- ✅ MX records configured correctly
- ✅ SPF record valid
- ✅ DKIM signing active
- ✅ DMARC policy set
- ✅ Reverse DNS (PTR) record matches mail server hostname
- ✅ Server IP not on common blocklists (Spamhaus, SORBS, etc.)

### Common Issues

<AccordionGroup>
  <Accordion title="Emails going to spam">
    Check SPF, DKIM, and DMARC are all configured. Ensure your server IP is not on spam blocklists. Check your sending reputation at [MXToolbox](https://mxtoolbox.com).
  </Accordion>

  <Accordion title="Cannot send email (port 25 blocked)">
    Most cloud providers block port 25 on new VPS instances. Contact your provider to unblock it, or use an SMTP relay service (SendGrid, SES) on port 587 instead.
  </Accordion>

  <Accordion title="TLS certificate errors">
    Ensure the SSL certificate covers the mail hostname (`mail.example.com`). Re-provision from **Email Server → SSL**.
  </Accordion>
</AccordionGroup>
