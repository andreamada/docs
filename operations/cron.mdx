## Creating a Cron Job

1. Go to **Cron Jobs** → **New Cron Job**
2. Select the target server
3. Configure:

| Field | Description |
|-------|-------------|
| **Name** | Descriptive label (for the UI only) |
| **Command** | Shell command to execute |
| **User** | System user to run the command as (default: `root` or `www-data`) |
| **Schedule** | When to run (see below) |
| **Working Directory** | Directory to `cd` into before running |
| **Timeout** | Kill the job if it runs longer than N minutes |
| **Enabled** | Toggle the job on/off without deleting it |

## Schedule Configuration

Use the visual schedule picker or enter a cron expression directly:

### Common Schedules

| Schedule | Cron Expression |
|----------|----------------|
| Every minute | `* * * * *` |
| Every 5 minutes | `*/5 * * * *` |
| Every hour | `0 * * * *` |
| Every day at 2am | `0 2 * * *` |
| Every Monday at midnight | `0 0 * * 1` |
| First of month at 3am | `0 3 1 * *` |

### Cron Expression Reference

```
┌─────────────── minute (0-59)
│  ┌──────────── hour (0-23)
│  │  ┌───────── day of month (1-31)
│  │  │  ┌────── month (1-12)
│  │  │  │  ┌─── day of week (0-7, 0 and 7 = Sunday)
│  │  │  │  │
*  *  *  *  *
```

<Tip>
Use [crontab.guru](https://crontab.guru) to test and verify cron expressions.
</Tip>

## WordPress Cron Jobs

AreaWP includes a built-in WordPress cron integration. Replace WP-Cron's default behavior (triggered by site visits) with a real server cron for better reliability:

```bash
# Run WordPress cron every 5 minutes
*/5 * * * * www-data /usr/local/bin/wp --path=/var/www/mysite.com cron event run --due-now --quiet
```

AreaWP can generate these commands automatically for each WordPress site — click **Add WP-Cron** from the WordPress site settings.

## Execution History

Every cron job execution is logged:

| Column | Description |
|--------|-------------|
| **Started at** | Timestamp when the job began |
| **Duration** | How long it ran |
| **Exit code** | `0` = success, non-zero = error |
| **Output** | stdout/stderr from the command |
| **Status** | ✅ Success / ❌ Failed / ⏱ Timeout |

Click any row to see the full command output.

## Notifications

Configure alerts for failed cron jobs:

1. Go to **Settings → Notifications → Cron**
2. Enable failure alerts
3. Set the alert channels (email, Slack, webhook)

<Info>
By default, AreaWP suppresses success notifications to avoid email flooding. Enable success notifications in settings if you need confirmation that critical jobs ran.
</Info>

## System Cron Jobs

Some AreaWP internal tasks also run as cron jobs. These are read-only in the UI:

| Job | Schedule | Purpose |
|-----|----------|---------|
| License check | Every 6 hours | Validate license with the license server |
| Certificate renewal check | Daily | Renew expiring SSL certificates |
| Backup execution | Per schedule | Run configured backup jobs |
| Monitor checks | Per interval | Run uptime monitor pings |
| Cleanup | Daily at 3am | Remove old logs and temp files |
