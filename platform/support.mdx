**Path:** Support (`/support`)

The Support section provides a built-in help center where clients can submit tickets and admins can manage support requests. It must be enabled in **Settings → Support Center**.
</Info>

## For Clients — Submitting a Ticket

### Creating a New Ticket

1. Navigate to **Support** in the sidebar
2. Click **New Ticket**
3. Fill in the ticket form:

| Field | Description |
|-------|-------------|
| **Subject** | Brief description of the issue |
| **Category** | Billing / Technical / Account / General |
| **Priority** | Low / Normal / High / Urgent |
| **Description** | Detailed explanation of the problem |
| **Attachments** | Screenshots, error logs, or other files |

4. Click **Submit** — you'll receive an email confirmation

### Tracking Your Ticket

The Support page shows all your tickets:

| Column | Description |
|--------|-------------|
| **Ticket #** | Unique identifier |
| **Subject** | Ticket title |
| **Status** | Open / In Progress / Waiting / Closed |
| **Priority** | Low / Normal / High / Urgent |
| **Last update** | When the ticket was last updated |

Click any ticket to read the conversation and add replies.

### Ticket Statuses

| Status | Meaning |
|--------|---------|
| 🔵 **Open** | Submitted, waiting for support response |
| 🟡 **In Progress** | Support team is working on it |
| 🟠 **Waiting for Client** | Support replied, waiting for your response |
| 🟢 **Closed** | Issue resolved |

Closed tickets can be reopened within 7 days by adding a reply.

## For Admins — Managing Support

### Ticket Queue

Admins see all tickets from all clients in the support queue:

| Column | Description |
|--------|-------------|
| **Client** | User who submitted the ticket |
| **Subject** | Ticket title |
| **Category** | Issue category |
| **Priority** | Urgency level |
| **Status** | Current state |
| **Agent** | Assigned support agent |
| **Created** | Submission date |
| **SLA** | Time remaining before SLA breach |

### Managing a Ticket

1. Click any ticket to open it
2. Read the full conversation
3. Assign to a support agent
4. Change priority or category
5. Add an internal note (not visible to client)
6. Reply to the client
7. Close when resolved

### Ticket Macros

Create saved responses for common issues:

1. Go to **Support → Macros** → **New Macro**
2. Name the macro (e.g. "DNS Propagation Explanation")
3. Write the template response (supports `{{client_name}}`, `{{site_domain}}` variables)
4. Save — use the macro in any ticket reply

### SLA Configuration

Set response time targets in **Settings → Support → SLA**:

| Priority | First Response | Resolution |
|----------|---------------|------------|
| **Urgent** | 1 hour | 4 hours |
| **High** | 4 hours | 24 hours |
| **Normal** | 8 hours | 48 hours |
| **Low** | 24 hours | 7 days |

Tickets approaching SLA breach are highlighted in the queue.

## Enabling the Support Center

The Support Center is disabled by default. To enable:

1. Go to **Administration → Settings**
2. Find **Support Center**
3. Toggle **Enable Support Center** to on
4. Configure notification email for new tickets
5. Save

The **Support** link will appear in the sidebar for all users.

<Tip>
Even if you use external support software (Zendesk, Freshdesk, Intercom), you can link clients to it by setting a **Support URL** in **Settings → General → Support URL**. The sidebar Support link will open your external tool instead.
</Tip>
