## Panel Server Requirements

The AreaWP panel runs on a dedicated Linux server or VPS.

<CardGroup cols={2}>
  <Card title="Minimum" icon="server" color="#f59e0b">
    1 vCPU · 1 GB RAM · 20 GB disk
  </Card>
  <Card title="Recommended" icon="server" color="#10b981">
    2 vCPU · 2 GB RAM · 40 GB disk
  </Card>
</CardGroup>

### Operating System

| OS | Version | Status |
|----|---------|--------|
| Ubuntu | 22.04 LTS | ✅ Fully supported |
| Ubuntu | 20.04 LTS | ✅ Supported |
| Debian | 11 / 12 | ✅ Supported |
| Fedora | 35+ | ✅ Supported |

<Warning>
AreaWP is designed for **Linux servers only**. Windows and macOS are supported for local development only (via WSL2).
</Warning>

### Software Dependencies

The installer handles all of the following automatically:

| Dependency | Version | Purpose |
|-----------|---------|---------|
| Python | 3.11 – 3.12 | Backend API (Flask) |
| Node.js | 20.x | Frontend build |
| PostgreSQL | 14+ | Panel database |
| Nginx | Latest | Reverse proxy |
| Docker | 24+ | App container runtime |
| Docker Compose | v2 plugin | Multi-container stacks |
| Gunicorn | 23+ | Production WSGI server |

## Managed Server Requirements

Servers you manage via AreaWP need only the lightweight agent binary installed.

| Requirement | Minimum |
|------------|---------|
| OS | Ubuntu 20.04+, Debian 11+, Fedora 35+ |
| RAM | 256 MB free |
| Disk | 1 GB free |
| Outbound port | 443 (to panel) |
| Docker | Optional — required for app deployments |

<Tip>
The agent is a single static Go binary (~15 MB) with no runtime dependencies. It runs as a systemd service.
</Tip>

## Network Requirements

### Panel Server

| Port | Purpose |
|------|---------|
| 80 | HTTP → HTTPS redirect |
| 443 | HTTPS panel access |
| 5000 | Internal backend API (local only) |

### Managed Servers

| Port | Purpose |
|------|---------|
| 22 | SSH for initial agent installation |
| 443 | Agent WebSocket connection to panel (outbound) |
| 80, 443 | Hosted websites |
| 8001–8999 | Docker app containers (internal) |

<Info>
The agent connects **outbound** to the panel. No inbound firewall rules are needed on managed servers for the agent itself.
</Info>
