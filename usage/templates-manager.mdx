## Overview

## Managing Templates

### Adding New Templates

1. Click **Add New Template** button
2. Choose template source:
   - **WordPress.org Repository:** Free themes from WordPress
   - **Upload Theme:** Upload custom theme ZIP file
   - **Premium Theme:** Add purchased premium themes
3. Enter theme details:
   - **Theme Name:** Display name for customers
   - **Theme Slug:** WordPress theme directory name
   - **Description:** Brief description of theme features
   - **Category:** Theme category (Business, Blog, Portfolio, etc.)
4. Upload theme preview image (recommended: 1200 x 900 px)
5. Click **Save** to add the template.
6. **Next Step:** Go to [Browse Templates](#template-ordering-and-visibility) to enable the theme and make it visible to customers.

<Tip>
**Theme Preview Images:** Use high-quality screenshots showing the theme's homepage. This helps customers visualize how their website will look.
</Tip>

### Template Properties

| Property | Description |
|----------|-------------|
| **Theme Name** | Display name shown to customers |
| **Theme Slug** | WordPress theme directory name (e.g., `twentytwentyfour`) |
| **Description** | Brief description of theme features and style |
| **Category** | Theme category for organization |
| **Preview Image** | Screenshot of theme (1200 x 900 px recommended) |
| **Status** | Managed via the [Browse Templates](#template-ordering-and-visibility) screen. |
| **Default** | Set as default theme for new websites |
| **Featured** | Highlight as recommended theme |
| **Enable Sync** | Enable template synchronization for automatic updates |

## Template Ordering and Visibility

While basic theme details are managed in the Template Manager, the **visibility** and **display order** are managed separately.

<Info>
**Navigation Path:** Admin Dashboard → Browse Templates  
**URL:** `/admin/WebsiteTemplateSort`
</Info>

### Managing Visibility

1. Navigate to the **Browse Templates** page.
2. Find the template you want to show or hide.
3. Toggle the **Visibility** switch to **Active** (available to customers) or **Inactive** (hidden).

### Reordering Templates

The order in which themes appear to customers during website creation can be customized:

1. On the **Browse Templates** page, locate the template cards.
2. Use **Drag and Drop** to rearrange the templates.
3. The new order is saved automatically and reflected in the customer dashboard instantly.

## Template Synchronization

Template Synchronization is a powerful feature that allows you to automatically push updates from a master template to all websites created from it.

### How Template Synchronization Works

When enabled, any changes made to the master template are automatically synchronized to all websites that:
1. Were created from this template
2. Have synchronization enabled on their website settings

**What Gets Synchronized:**
- **Theme Files:** Template files, stylesheets, and assets
- **Plugin Changes:** Plugin installations, updates, and removals
- **Configuration:** Theme settings and customizations
- **Content Updates:** Template pages and posts (optional)
- **Widget Configurations:** Sidebar and widget settings

<Info>
**New Feature:** Template Synchronization helps you maintain consistency across multiple websites and deploy updates efficiently without manually updating each site.
</Info>

### Enabling Template Synchronization

**On the Template:**

1. Edit the template in Template Manager
2. Enable **Template Synchronization** toggle
3. Configure sync settings:
   - **Sync Frequency:** Real-time, Hourly, Daily, Manual
   - **Sync Scope:** Files only, Files + Plugins, Full sync
   - **Backup Before Sync:** Create backup before applying changes
4. Save template settings

**On Individual Websites:**

Customers or admins can enable/disable sync per website:
1. Go to website settings
2. Enable **Sync with Template** option
3. Website will receive template updates automatically

<Tip>
**Selective Sync:** Websites can opt-out of synchronization if they need custom modifications. This gives flexibility while maintaining the option to sync when needed.
</Tip>

### Sync Modes

<Tabs>
  <Tab title="Real-Time Sync">
    **When to Use:** Critical updates, small changes
    
    - Changes sync immediately after saving template
    - Websites update within minutes
    - Best for urgent security fixes
    - Higher server resource usage
  </Tab>
  
  <Tab title="Scheduled Sync">
    **When to Use:** Regular updates, large changes
    
    - Sync runs at scheduled intervals (hourly, daily)
    - Lower server resource usage
    - Predictable update windows
    - Better for batch changes
  </Tab>
  
  <Tab title="Manual Sync">
    **When to Use:** Major updates, testing required
    
    - Admin triggers sync manually
    - Full control over update timing
    - Test changes before deployment
    - Best for major template overhauls
  </Tab>
</Tabs>

## Theme Management

### Editing Templates

1. Find the theme in Template Manager
2. Click **Edit** button
3. Update theme details:
   - Name, description, category
   - Preview image
4. Save changes

### Deactivating Themes

To hide a theme from customers without deleting it:

1. Navigate to [Browse Templates](#template-ordering-and-visibility).
2. Find the theme to deactivate.
3. Toggle status to **Inactive**.
4. Theme is hidden from customer selection; existing websites using it are unaffected.

<Info>
**Deactivation vs. Deletion:** Deactivating a theme hides it from new selections but doesn't affect existing websites. Deleting a theme can break existing sites using it.
</Info>

### Deleting Themes

1. Ensure no active websites are using the theme.
2. Find the theme in Template Manager.
3. Click **Delete** button.
4. Confirm deletion; theme files are removed from server.

<Warning>
**Before Deleting:** Check if any customer websites are using the theme. Deleting a theme in use can break those websites. Deactivate via the Browse Templates page instead if unsure.
</Warning>

## Best Practices

<AccordionGroup>
  <Accordion title="Theme Selection" icon="check">
    - Offer 10-20 quality themes, not hundreds.
    - Include themes for different purposes (business, blog, portfolio).
    - Choose well-maintained, regularly updated themes.
    - Prioritize mobile-responsive themes.
    - Select themes with good performance scores.
  </Accordion>
  
  <Accordion title="Theme Organization" icon="folder">
    - Categorize themes logically in Template Manager.
    - Feature your best 3-5 themes.
    - Change display order via **Browse Templates** to highlight top choices.
    - Update theme previews regularly.
  </Accordion>
  
  <Accordion title="Maintenance & Synchronization" icon="wrench">
    - Test template changes on a staging site before syncing.
    - Enable automatic backups before major sync operations.
    - Monitor sync logs for potential conflicts or failures.
    - Keep themes and plugins within templates updated.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Theme Not Appearing for Customers" icon="eye-slash">
    **Issue:** Theme is saved in Template Manager but not visible to customers.
    
    **Solutions:**
    1. Verify visibility is set to **Active** in [Browse Templates](#template-ordering-and-visibility).
    2. Check if the theme is in the correct category.
    3. Clear application/server cache.
  </Accordion>
  
  <Accordion title="Sync Not Working" icon="circle-xmark">
    **Issue:** Changes not syncing to websites.
    
    **Solutions:**
    1. Verify template sync is enabled in Template Manager correctly.
    2. Check website has sync enabled in its own settings.
    3. Review sync logs for errors.
    4. Verify cron jobs are running on the server.
  </Accordion>
  
  <Accordion title="Theme Upload Fails" icon="upload">
    **Issue:** Cannot upload custom theme ZIP file.
    
    **Solutions:**
    1. Check file size against server upload limits.
    2. Verify ZIP file is a valid WordPress theme structure.
    3. Ensure server directory has write permissions.
  </Accordion>
</AccordionGroup>
