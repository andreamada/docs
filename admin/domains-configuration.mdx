## Overview

The Domain API Settings page allows you to configure automatic domain registration through Dynadot's API. Set up nameservers, CNAME records, and manage which domain extensions (TLDs) are available for registration on your platform.

## Dynadot API Configuration

Connect your Dynadot account to enable automatic domain registration.

### API Token Setup

**Dynadot API Token**

Your API key from Dynadot for domain registration and management.

<Steps>
  <Step title="Log in to Dynadot">
    Visit [Dynadot.com](https://www.dynadot.com) and log in to your account
  </Step>
  <Step title="Access API Settings">
    Navigate to **My Account → API Settings** or visit [API Settings Page](https://www.dynadot.com/account/domain/setting/api.html)
  </Step>
  <Step title="Generate API Key">
    Create a new API key or copy your existing key
  </Step>
  <Step title="Enter Token">
    Paste the API token in the Domain API Settings page
  </Step>
  <Step title="Save Configuration">
    Save your settings to enable domain registration
  </Step>
</Steps>

<Warning>
**Security Important:**
- Keep your API token confidential
- Never share your API token publicly
- Regenerate the token if it's ever compromised
- Use IP restrictions in Dynadot if available
</Warning>

## Nameserver Configuration

Set the default nameservers that will be assigned to newly registered domains.

### Default Nameservers

**Primary Nameserver (NS1)**

First nameserver for domain DNS resolution.

- **Format**: `ns1.yourdomain.com` or IP address
- **Example**: `ns1.example.com`
- **Required**: Yes

**Secondary Nameserver (NS2)**

Second nameserver for redundancy and reliability.

- **Format**: `ns2.yourdomain.com` or IP address
- **Example**: `ns2.example.com`
- **Required**: Yes

### CNAME Record Domain

**Record Domain for CNAME**

Domain used for CNAME records when pointing custom domains to your platform.

- **Format**: `yourdomain.com` (without www)
- **Example**: `platform.example.com`
- **Purpose**: Allows users to point their domains to your service

<Info>
**How Nameservers Work:**

When a domain is registered through your platform:
1. Domain is registered via Dynadot API
2. Nameservers (NS1 and NS2) are automatically configured
3. DNS records point to your hosting infrastructure
4. Domain becomes active and accessible
</Info>

<Tip>
**Nameserver Best Practices:**
- Use reliable nameserver providers
- Ensure both NS1 and NS2 are on different servers for redundancy
- Test nameserver configuration before going live
- Keep nameserver settings consistent across your platform
</Tip>

## TLD Management

Control which domain extensions (Top-Level Domains) are available for registration.

### Domain Check Configuration

**Default Domain Check TLDs**

Domain extensions shown by default in the domain search/availability checker.

**Format**: One TLD per line (without the dot)

**Example:**
```
com
net
org
io
co
```

**Common TLDs:**
- `com` - Commercial (most popular)
- `net` - Network
- `org` - Organization
- `io` - Input/Output (popular for tech)
- `co` - Company
- `info` - Information
- `biz` - Business

### Extended Domain Check

**Extended Domain Check TLDs**

Additional domain extensions shown when users click "Show More" or expand options.

**Format**: One TLD per line (without the dot)

**Example:**
```
app
dev
tech
online
store
shop
blog
site
website
cloud
```

**Popular Extended TLDs:**
- `app` - Applications
- `dev` - Developers
- `tech` - Technology
- `online` - Online presence
- `store` - E-commerce
- `shop` - Shopping
- `blog` - Blogging
- `site` - Websites
- `cloud` - Cloud services

<Info>
**TLD Selection Strategy:**

**Default TLDs** (5-10 options):
- Most popular and recognizable extensions
- Lower cost options
- General-purpose domains

**Extended TLDs** (10-30 options):
- Niche-specific extensions
- Premium or specialty TLDs
- Industry-specific options
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Domain Registration Fails" icon="circle-xmark">
    **Issue:** Domains cannot be registered through the platform
    
    **Common Causes:**
    - Invalid or expired API token
    - Insufficient Dynadot account balance
    - Domain already registered
    - API rate limits exceeded
    
    **Solutions:**
    1. Verify API token is correct and active
    2. Check Dynadot account balance
    3. Confirm domain availability
    4. Review API usage limits
    5. Check Dynadot API status
  </Accordion>
  
  <Accordion title="Nameservers Not Working" icon="server">
    **Issue:** Domains not resolving to correct nameservers
    
    **Common Causes:**
    - Incorrect nameserver configuration
    - DNS propagation delay
    - Nameserver not responding
    - Typo in nameserver address
    
    **Solutions:**
    1. Verify nameserver addresses are correct
    2. Wait 24-48 hours for DNS propagation
    3. Test nameserver response with dig/nslookup
    4. Check nameserver provider status
    5. Verify DNS records are configured
  </Accordion>
  
  <Accordion title="CNAME Records Not Resolving" icon="link">
    **Issue:** Custom domains not pointing correctly
    
    **Common Causes:**
    - Incorrect CNAME domain setting
    - DNS not propagated
    - CNAME conflicts with A records
    - Domain not configured properly
    
    **Solutions:**
    1. Verify CNAME domain is correct
    2. Wait for DNS propagation (up to 48 hours)
    3. Remove conflicting A records
    4. Check DNS configuration at domain registrar
    5. Test with DNS lookup tools
  </Accordion>
  
  <Accordion title="TLDs Not Appearing" icon="list">
    **Issue:** Domain extensions not showing in search
    
    **Common Causes:**
    - TLDs not saved properly
    - Cache not cleared
    - Incorrect TLD format
    - Configuration not applied
    
    **Solutions:**
    1. Re-enter TLDs (one per line, no dots)
    2. Clear application cache
    3. Verify TLD format is correct
    4. Save configuration again
    5. Refresh the domain search page
  </Accordion>
  
  <Accordion title="API Token Invalid" icon="key">
    **Issue:** API token shows as invalid or unauthorized
    
    **Common Causes:**
    - Token expired or revoked
    - Token copied incorrectly
    - IP restrictions in Dynadot
    - Account suspended
    
    **Solutions:**
    1. Generate new API token in Dynadot
    2. Copy token carefully (no extra spaces)
    3. Check IP restriction settings
    4. Verify Dynadot account is active
    5. Contact Dynadot support if needed
  </Accordion>
</AccordionGroup>

```bash
# Check nameserver response
nslookup domain.com ns1.yourplatform.com

# Detailed DNS query
dig @ns1.yourplatform.com domain.com

# Check nameserver propagation
host domain.com
```
