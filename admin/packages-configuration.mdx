## Website Packages

Create and configure hosting packages with customizable features, pricing, and resource allocations.

### Package Configuration Fields

<ParamField path="name" type="string" required>
  **Package Name**
  
  Display name for the hosting package.
  
  - **Example:** `Startup`, `Business`, `Professional`, `Advanced`
  - **Purpose:** Shown on pricing pages and customer dashboards
  - **Best Practice:** Use clear, tier-based naming
</ParamField>

<ParamField path="price" type="currency" required>
  **Price / Month**
  
  Monthly subscription price for the package.
  
  - **Format:** Currency value
  - **Note:** Yearly price calculated automatically based on renewal terms
  - **Example:** `$9.99`, `$19.99`, `$49.99`
</ParamField>

<ParamField path="disk" type="number" required>
  **Storage (MB)**
  
  Disk space allocation for the package.
  
  - **Min:** 150 MB
  - **Max:** 99,999,999,999 MB
  - **Example:** `1000` (1 GB), `5000` (5 GB), `10000` (10 GB)
  - **Recommendation:** Start with 1-5 GB for basic plans
</ParamField>

<ParamField path="description" type="textarea">
  **Description**
  
  Detailed description of package features and benefits.
  
  - **Format:** Multi-line text
  - **Purpose:** Displayed on pricing pages
  - **Best Practice:** Highlight key features and differentiators
  - **Example:** "Perfect for small businesses. Includes SSL, daily backups, and 24/7 support."
</ParamField>

<ParamField path="roles" type="checkbox">
  **Member Roles**
  
  User roles that can purchase this package.
  
  - **Options:** Customer roles defined in system
  - **Default:** `c1` (standard customer)
  - **Use Case:** Restrict premium packages to verified users
</ParamField>

<ParamField path="background_color1" type="color" required>
  **Background Color 1**
  
  Primary gradient color for package card.
  
  - **Format:** Hex color code
  - **Default:** `#FF00CC`
  - **Purpose:** Visual branding on pricing page
</ParamField>

<ParamField path="background_color2" type="color" required>
  **Background Color 2**
  
  Secondary gradient color for package card.
  
  - **Format:** Hex color code
  - **Default:** `#FF00CC`
  - **Purpose:** Creates gradient effect with Color 1
</ParamField>

### Package Feature Toggles

<ParamField path="enable_purchase_theme" type="switch">
  **Enable Purchase Theme Option**
  
  Allow customers to purchase premium themes.
  
  - **Default:** Off
  - **Impact:** Shows theme marketplace in customer dashboard
  - **Revenue:** Additional upsell opportunity
</ParamField>

<ParamField path="enable_backup" type="switch">
  **Enable Backup Download Option**
  
  Allow customers to download website backups.
  
  - **Default:** Off
  - **Pricing:** Configure in Global Service Pricing section
  - **Note:** If enabled, set backup price in right panel
</ParamField>

<ParamField path="enable_directadmin" type="switch">
  **Enable Control Panel & PhpMyAdmin Access**
  
  Grant access to DirectAdmin control panel and database management.
  
  - **Default:** Off
  - **Use Case:** Advanced users who need direct server access
  - **Security:** Only enable for trusted user tiers
</ParamField>

<ParamField path="suggest" type="switch">
  **Featured Ribbon on Pricing Page**
  
  Display "Featured" or "Most Popular" badge on this package.
  
  - **Default:** Off
  - **Purpose:** Highlight recommended package
  - **Best Practice:** Feature your most profitable or popular plan
</ParamField>

## Renewal Terms

Configure billing cycles and discount structures for long-term subscriptions.

### Renewal Configuration Fields

<ParamField path="name" type="string" required>
  **Renewal Term Name**
  
  Display name for the billing cycle.
  
  - **Example:** `1 Month`, `3 Months`, `6 Months`, `1 Year`
  - **Purpose:** Shown during checkout
</ParamField>

<ParamField path="days" type="number" required>
  **Duration (Days)**
  
  Length of the billing cycle in days.
  
  - **Min:** 1 day
  - **Max:** 99,999,999,999 days
  - **Common Values:**
    - 30 days = 1 month
    - 90 days = 3 months
    - 180 days = 6 months
    - 365 days = 1 year
</ParamField>

<ParamField path="sale_offer" type="number">
  **Discount Percentage**
  
  Discount applied for this billing cycle.
  
  - **Min:** 0%
  - **Max:** 90%
  - **Purpose:** Incentivize longer commitments
  - **Example:** 10% for 6 months, 20% for 1 year
</ParamField>

<ParamField path="roles" type="checkbox">
  **Available For**
  
  User roles that can select this renewal term.
  
  - **Default:** `c1` (all customers)
  - **Use Case:** Offer exclusive discounts to premium members
</ParamField>

### Renewal Term Examples

<CodeGroup>
```yaml Monthly Billing
Name: 1 Month
Days: 30
Discount: 0%
Available For: All customers

Calculation:
- Package Price: $19.99/month
- Total: $19.99 (no discount)
```

```yaml Quarterly Billing
Name: 3 Months
Days: 90
Discount: 10%
Available For: All customers

Calculation:
- Package Price: $19.99/month
- Subtotal: $19.99 × 3 = $59.97
- Discount: $59.97 × 10% = $5.99
- Total: $53.98 (save $5.99)
```

```yaml Semi-Annual Billing
Name: 6 Months
Days: 180
Discount: 15%
Available For: All customers

Calculation:
- Package Price: $19.99/month
- Subtotal: $19.99 × 6 = $119.94
- Discount: $119.94 × 15% = $17.99
- Total: $101.95 (save $17.99)
```

```yaml Annual Billing
Name: 1 Year
Days: 365
Discount: 20%
Available For: All customers

Calculation:
- Package Price: $19.99/month
- Subtotal: $19.99 × 12 = $239.88
- Discount: $239.88 × 20% = $47.98
- Total: $191.90 (save $47.98)
```
</CodeGroup>

<Tip>
**Discount Strategy Tips:**
- **Monthly:** 0% discount (baseline pricing)
- **Quarterly:** 5-10% discount
- **Semi-Annual:** 10-15% discount
- **Annual:** 15-25% discount
- **Biennial:** 25-30% discount (for enterprise)
</Tip>

## Global Service Pricing

Configure pricing for additional services available across all packages.

### Service Pricing Fields

<ParamField path="ssl_price" type="currency">
  **SSL Certificate Price**
  
  Global pricing for SSL certificates.
  
  - **Min:** $0
  - **Max:** $99,999
  - **Default:** $0 (free SSL via Let's Encrypt)
  - **Use Case:** Premium SSL certificates (EV, Wildcard)
</ParamField>

<ParamField path="backup_price" type="currency">
  **Backup Download Price**
  
  Price for backup download service.
  
  - **Min:** $0
  - **Max:** $99,999
  - **Default:** $0
  - **Important:** Configure per-package in left panel if enabled
  - **Note:** Highlighted field with special styling
</ParamField>

<ParamField path="upgrade_disk_price" type="currency">
  **Disk Upgrade Price per MB**
  
  Cost per MB for additional storage.
  
  - **Min:** $0
  - **Max:** $99,999
  - **Default:** $2,000 (currency units)
  - **Example:** $0.01/MB = $10/GB
</ParamField>

<Warning>
**Backup Pricing Important Note:**

If you set a Backup Price, you MUST configure the backup option for each package individually in the Website Packages section. Enable the "Enable Backup Download Option" switch for packages that should include this feature.
</Warning>

### Service Pricing Examples

```yaml Free SSL Model
Strategy:
```

```yaml Premium Services Model
- Charge for extra storage
```


## Domain Pricing

Set custom pricing for domain registration and renewal by TLD (Top Level Domain).

### Domain Pricing Configuration

<Info>
**Pricing Override:** Leaving the price blank will use the original price from Dynadot. Only fill in prices you want to customize.
</Info>

### Pricing Fields

For each TLD, you can configure:

<ParamField path="reg_price" type="currency">
  **Sale Price (without VAT)**
  
  Your custom registration price before tax.
  
  - **Default:** Uses Dynadot price
  - **Purpose:** Set your markup or discount
  - **Example:** Dynadot: $10.99 → Your Price: $12.99
</ParamField>

<ParamField path="reg_price_vat" type="currency">
  **Sale Price (with VAT)**
  
  Registration price including tax.
  
  - **Default:** Uses Dynadot price + VAT
  - **Calculation:** Typically reg_price × (1 + VAT rate)
  - **Example:** $12.99 × 1.10 = $14.29 (10% VAT)
</ParamField>

<ParamField path="renew_price" type="currency">
  **Renewal Price**
  
  Annual renewal price for the domain.
  
  - **Default:** Uses Dynadot renewal price
  - **Strategy:** Can be same as or different from registration
  - **Common:** Higher renewal than registration (promotional pricing)
</ParamField>

## Custom Addons

Create additional services and upsell opportunities for your customers.

### Addon Configuration Fields

<ParamField path="name" type="string" required>
  **Addon Name**
  
  Display name for the addon service.
  
  - **Example:** `Priority Support`, `SEO Optimization`, `Content Writing`
  - **Purpose:** Shown during checkout and in customer dashboard
</ParamField>

<ParamField path="addon_term" type="select" required>
  **Payment Term**
  
  Billing frequency for the addon.
  
  - **Options:**
    - `One Time` - Single payment
    - `Monthly` - Recurring monthly charge
    - `Yearly` - Recurring annual charge
  - **Default:** One Time
</ParamField>

<ParamField path="addon_price" type="currency" required>
  **Addon Price**
  
  Price for the addon service.
  
  - **Min:** Must be greater than 0
  - **Format:** Currency value
  - **Example:** `$49.99`, `$99.99`, `$199.99`
</ParamField>

<ParamField path="linked_packages" type="checkbox">
  **Attach to Packages**
  
  Select which hosting packages can purchase this addon.
  
  - **Options:** All created packages
  - **Use Case:** Restrict premium addons to higher-tier plans
  - **Example:** SEO service only for Business+ packages
</ParamField>

<ParamField path="addon_description" type="textarea">
  **Description**
  
  Detailed description of the addon service.
  
  - **Format:** Multi-line text
  - **Purpose:** Explain value proposition
  - **Best Practice:** Highlight benefits and deliverables

## Troubleshooting

<AccordionGroup>
  <Accordion title="Package Not Showing on Pricing Page" icon="eye-slash">
    **Common Issues:**
    - Package not saved properly
    - Member role restrictions
    - Package disabled or deleted
    - Cache not cleared
    
    **Solutions:**
    - Verify package is saved (click Save Configuration)
    - Check member role settings (should include target users)
    - Ensure package is active
    - Clear browser and server cache
    - Check frontend template for errors
  </Accordion>
  
  <Accordion title="Renewal Discounts Not Calculating" icon="percent">
    **Common Issues:**
    - Discount percentage set incorrectly
    - Days calculation wrong
    - Role restrictions preventing access
    - JavaScript errors on checkout
    
    **Solutions:**
    - Verify discount is 0-90%
    - Check days match intended period (30, 90, 365)
    - Ensure user role has access to term
    - Check browser console for errors
    - Test calculation manually
  </Accordion>
  
  <Accordion title="Backup Price Not Applying" icon="database">
    **Common Issues:**
    - Global backup price set but package option disabled
    - Package option enabled but no global price
    - Conflicting pricing settings
    
    **Solutions:**
    - Set global backup price in right panel
    - Enable "Backup Download Option" for each package
    - Verify both settings are configured
    - Check package-specific pricing overrides
    - Review customer billing details
  </Accordion>
  
  <Accordion title="Domain Pricing Not Updating" icon="globe">
    **Common Issues:**
    - Dynadot API not configured
    - Price fields left blank (using default)
    - VAT calculation incorrect
    - Cache not cleared
    
    **Solutions:**
    - Configure Dynadot API in Domain API Settings
    - Enter custom prices (blank = use Dynadot price)
    - Verify VAT rate in system settings
    - Clear pricing cache
    - Test with different TLDs
  </Accordion>
  
  <Accordion title="Addon Not Available for Package" icon="plus-square">
    **Common Issues:**
    - Addon not linked to package
    - Package restrictions
    - Addon price set to 0
    - Addon disabled
    
    **Solutions:**
    - Check "Attach to Packages" settings
    - Verify package is in linked list
    - Ensure addon price > 0
    - Confirm addon is active
    - Test with different packages
  </Accordion>
</AccordionGroup>
