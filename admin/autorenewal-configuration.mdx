## Website Auto-Renewal

Configure automatic renewal settings for website hosting packages.

### Enable Website Auto-Renewal

**Auto-Renewal Status**

Master toggle to enable or disable automatic website renewal across the platform.

- **Enabled**: Websites automatically renew before expiration
- **Disabled**: Users must manually renew their websites

### How Website Auto-Renewal Works

<Steps>
  <Step title="Renewal Date Approaches">
    System detects upcoming website expiration (typically 7-14 days before)
  </Step>
  <Step title="Payment Attempt">
    Automatic payment is processed using saved payment method
  </Step>
  <Step title="Renewal Processed">
    If payment succeeds, website is renewed for the next billing period
  </Step>
  <Step title="Confirmation Sent">
    Customer receives email confirmation of successful renewal
  </Step>
  <Step title="Service Continues">
    Website remains active without interruption
  </Step>
</Steps>

### Website Renewal Settings

| Setting | Description | Recommended |
|---------|-------------|-------------|
| **Auto-Renewal** | Enable automatic website renewal | Enabled |
| **Renewal Notice** | Days before expiration to send reminder | 7-14 days |
| **Retry Failed Payments** | Attempt payment again if first attempt fails | Yes |
| **Retry Attempts** | Number of payment retry attempts | 2-3 times |
| **Retry Interval** | Days between retry attempts | 2-3 days |

<Tip>
**Website Auto-Renewal Benefits:**
- Reduces customer churn
- Ensures continuous service
- Minimizes manual processing
- Improves customer satisfaction
- Provides predictable revenue
</Tip>

## Domain Auto-Renewal

Configure automatic renewal settings for domain registrations.

### Enable Domain Auto-Renewal

**Auto-Renewal Status**

Master toggle to enable or disable automatic domain renewal across the platform.

- **Enabled**: Domains automatically renew before expiration
- **Disabled**: Users must manually renew their domains

### How Domain Auto-Renewal Works

<Steps>
  <Step title="Renewal Date Approaches">
    System detects upcoming domain expiration (typically 30 days before)
  </Step>
  <Step title="Payment Attempt">
    Automatic payment is processed using saved payment method
  </Step>
  <Step title="Domain Renewed">
    If payment succeeds, domain is renewed through registrar API
  </Step>
  <Step title="Confirmation Sent">
    Customer receives email confirmation with new expiration date
  </Step>
  <Step title="Domain Remains Active">
    Domain continues to function without interruption
  </Step>
</Steps>

### Domain Renewal Settings

| Setting | Description | Recommended |
|---------|-------------|-------------|
| **Auto-Renewal** | Enable automatic domain renewal | Enabled |
| **Renewal Notice** | Days before expiration to send reminder | 30 days |
| **Early Renewal** | Days before expiration to process renewal | 7-14 days |
| **Grace Period** | Days after expiration before domain is released | 30 days |
| **Retry Failed Payments** | Attempt payment again if first attempt fails | Yes |

<Warning>
**Domain Expiration Risks:**
- Expired domains may be lost permanently
- Domain can be registered by someone else
- Email and website services will stop working
- Recovery can be expensive or impossible

**Always enable auto-renewal for critical domains!**
</Warning>

## Renewal Notifications

Configure email notifications for upcoming renewals.

### Notification Schedule

**Website Renewal Reminders:**
- **30 days before**: First reminder
- **14 days before**: Second reminder
- **7 days before**: Final reminder
- **Day of expiration**: Urgent notice

**Domain Renewal Reminders:**
- **60 days before**: Early notice
- **30 days before**: First reminder
- **14 days before**: Second reminder
- **7 days before**: Final reminder
- **Day of expiration**: Urgent notice

### Notification Content

Each renewal reminder should include:
- Service name (website or domain)
- Current expiration date
- Renewal amount
- Payment method on file
- Link to manage auto-renewal settings
- Instructions for manual renewal

<Tip>
**Notification Best Practices:**
- Send multiple reminders at different intervals
- Make renewal process easy with direct links
- Clearly state consequences of non-renewal
- Provide options to update payment methods
- Include customer support contact information
</Tip>

## Payment Processing

Configure how automatic renewal payments are handled.

### Payment Methods

**Supported Payment Methods for Auto-Renewal:**
- Credit/Debit Cards (via Stripe)
- PayPal (with billing agreement)
- Saved payment methods

**Not Supported:**
- Bank transfers (manual only)
- Cash payments
- One-time payment methods

### Failed Payment Handling

**When a Renewal Payment Fails:**

<Steps>
  <Step title="Initial Failure">
    First payment attempt fails (card declined, insufficient funds, etc.)
  </Step>
  <Step title="Customer Notification">
    Immediate email notification about failed payment
  </Step>
  <Step title="Retry Attempts">
    System retries payment after configured interval (e.g., 2-3 days)
  </Step>
  <Step title="Multiple Retries">
    Up to 3 retry attempts over 7-10 days
  </Step>
  <Step title="Final Notice">
    If all attempts fail, send final notice before service suspension
  </Step>
  <Step title="Grace Period">
    Allow grace period for manual payment before termination
  </Step>
</Steps>

### Payment Retry Configuration

| Setting | Recommended Value |
|---------|-------------------|
| **Retry Attempts** | 3 times |
| **First Retry** | 2 days after initial failure |
| **Second Retry** | 5 days after initial failure |
| **Third Retry** | 7 days after initial failure |
| **Grace Period** | 7-14 days after final failure |

<Info>
**Payment Failure Reasons:**
- Expired credit card
- Insufficient funds
- Card declined by bank
- Payment method removed
- Billing address mismatch
- Card security check failed
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Auto-Renewal Not Working" icon="circle-xmark">
    **Issue:** Services not renewing automatically
    
    **Common Causes:**
    - Auto-renewal disabled in settings
    - Cron jobs not running
    - Payment gateway issues
    - Customer disabled auto-renewal
    
    **Solutions:**
    1. Verify auto-renewal is enabled in settings
    2. Check cron jobs are running properly
    3. Test payment gateway connectivity
    4. Review customer account settings
    5. Check system logs for errors
  </Accordion>
  
  <Accordion title="Payment Failures" icon="credit-card">
    **Issue:** Renewal payments consistently failing
    
    **Common Causes:**
    - Expired payment methods
    - Insufficient funds
    - Payment gateway configuration
    - Card security issues
    
    **Solutions:**
    1. Contact customer to update payment method
    2. Verify payment gateway is configured correctly
    3. Check for payment processor issues
    4. Review transaction logs
    5. Test with a different payment method
  </Accordion>
  
  <Accordion title="Notifications Not Sending" icon="bell">
    **Issue:** Renewal reminders not being sent
    
    **Common Causes:**
    - Email service not configured
    - Cron jobs not running
    - Email templates missing
    - Customer email invalid
    
    **Solutions:**
    1. Verify email service is configured
    2. Check cron job execution
    3. Review email templates
    4. Validate customer email addresses
    5. Check spam/junk folders
  </Accordion>
  
  <Accordion title="Domains Expiring Despite Auto-Renewal" icon="link">
    **Issue:** Domains expiring even with auto-renewal enabled
    
    **Common Causes:**
    - Domain API issues
    - Insufficient registrar account balance
    - Domain locked or on hold
    - Renewal timing too late
    
    **Solutions:**
    1. Check domain registrar API status
    2. Verify registrar account has sufficient balance
    3. Ensure domain is unlocked
    4. Adjust renewal timing to earlier
    5. Contact domain registrar support
  </Accordion>
  
  <Accordion title="Customers Charged Incorrectly" icon="dollar-sign">
    **Issue:** Wrong amounts charged for renewals
    
    **Common Causes:**
    - Pricing not updated
    - Tax calculation errors
    - Discount codes not applied
    - Currency conversion issues
    
    **Solutions:**
    1. Verify current pricing is correct
    2. Check tax calculation settings
    3. Review discount code logic
    4. Confirm currency settings
    5. Issue refunds for overcharges
  </Accordion>
</AccordionGroup>
