## Overview

The General Settings page is a comprehensive admin interface for configuring system-wide settings, payment gateways, company information, and integrations. The page features a tabbed interface with six main sections.

---

## Navigation Tabs

The page includes the following tabs:
- **Contact Button** - Configure contact forms and communication channels
- **Company Setup** - Manage company information and website settings
- **Payment Gateway** - Configure payment methods and integrations
- **License Settings** - Manage software licensing
- **Cron Automation** - Set up automated tasks
- **Language Settings** - Configure multi-language support

### Features

#### Contact Form Settings
- **Enable Contact Form Request Callback** - Toggle to allow visitors to request callbacks through a contact form
- **Enable Helpdesk & Tickets System** - Toggle to enable the helpdesk ticketing system for customer support (visible in admin menu when enabled)

#### Contact Buttons Management
A dynamic item manager that allows you to create multiple contact buttons with the following fields:
- **Button Title** - Display name (e.g., Facebook, WhatsApp, Telegram)
- **Button Icon** - Upload custom icon image
- **Contact Link** - URL for the contact method (e.g., https://wa.me/1234567890)
- **Background Color** - Custom color picker for button styling
- **Open in New Tab** - Toggle to control link behavior

**Maximum:** 20 contact buttons  
**Features:** Sortable, drag-and-drop interface

#### Sidebar Guide
The right sidebar provides helpful information:
- **Popular Contact Options** with example URLs:
  - WhatsApp: `https://wa.me/1234567890`
  - Facebook Messenger: `https://m.me/username`
  - Telegram: `https://t.me/username`
  - Email: `mailto:contact@domain.com`

- **Best Practices:**
  - Use recognizable brand colors
  - Keep button titles short and clear
  - Test all links before going live
  - Limit to 3-5 contact options

## 3. Payment Gateway Tab

![Payment Gateway Configuration](C:/Users/mada/.gemini/antigravity/brain/cfd041e8-6dde-4395-b5b5-24b1fb67b901/payment_gateway_tab_1763752316931.png)

### Bank Account Information

A dynamic item manager for adding multiple bank accounts with:
- **Bank Logo** - Upload bank logo image
- **Bank Name** - Name of financial institution (e.g., Vietinbank, Chase Bank)
- **Account Holder Name** - Full legal name on account
- **Account Number** - Bank account number
- **Bank Branch** - Branch location or address

**Maximum:** 20 bank accounts  
**Features:** Sortable interface

### Stripe Payment Gateway

- **Enable Stripe Payment Processing** - Master toggle
- **Display Name** - Custom label (default: "Credit Card")
- **Publishable Key** - Public API key (pk_live_... or pk_test_...)
- **Secret Key** - Private API key (sk_live_... or sk_test_...)
- **Webhook Secret** - Webhook endpoint secret (whsec_...)

> [!NOTE]
> Configure your Stripe API keys to enable secure credit card processing.

### PayPal Payment Gateway

- **Enable PayPal Payment Processing** - Master toggle
- **Display Name** - Custom label (default: "PayPal")
- **Environment Mode** - Choose between:
  - Sandbox (Testing)
  - Live (Production)
- **Client ID** - PayPal application client ID
- **Client Secret** - PayPal application secret key

> [!NOTE]
> Configure PayPal credentials to enable PayPal payment processing.

## Technical Details

### File Structure
- **Main Template:** `GeneralOption.blade.php` (3,585 lines)
- **JavaScript:** `/assets/admin/js/settings.js`
- **Form Processing:** POST method with storage arrays

### Data Storage
Settings are stored using the `Storage` facade with two main types:
- `storage[setting][key]` - General settings
- `storage[option][key]` - System options

### AJAX Functionality
The page includes AJAX handlers for:
- License validation (`validate_license`)
- Website count retrieval (`get_website_count`)

### Security Features
- CSRF protection via form tokens
- Password fields for sensitive data (API keys, secrets)
- Input validation and sanitization
- Error handling with detailed logging

## Form Submission

All settings are saved via a single form submission using the POST method. The form encompasses all tabs, allowing users to modify settings across multiple sections before saving.

**Save Button Location:** Typically at the bottom of the page or in a fixed footer bar.

## Best Practices

> [!TIP]
> **Configuration Tips:**
> - Always test payment gateways in sandbox/test mode before going live
> - Keep API keys and secrets secure - never share them publicly
> - Regularly validate your license to ensure continued support
> - Configure at least one contact method for customer communication
> - Enable tax management if required by your jurisdiction

> [!WARNING]
> **Important Considerations:**
> - Changing currency settings may affect existing pricing
> - Disabling user registration will prevent new account creation
> - Payment gateway changes require thorough testing
> - Cron automation requires proper server configuration
