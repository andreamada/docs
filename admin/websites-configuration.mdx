## Resource Allocation

Configure the default disk space and bandwidth allocated to each website.

### Disk Space Settings

**Default Disk Space**

Amount of disk storage allocated to each website (in MB).

- **Minimum**: 150 MB
- **Maximum**: 99,999,999,999 MB
- **Recommended**: 1,000 - 5,000 MB for standard websites

### Bandwidth Settings

**Monthly Bandwidth Limit**

Amount of data transfer allowed per month (in MB).

- **Minimum**: 150 MB
- **Maximum**: 99,999,999,999 MB
- **Recommended**: 10,000 - 50,000 MB depending on traffic expectations

**Bandwidth Warning Threshold**

Percentage at which users receive a warning about approaching their bandwidth limit.

- **Range**: 0 - 100%
- **Recommended**: 80% (users get notified at 80% usage)

<Tip>
**Resource Planning Tips:**
- Start with conservative limits and increase based on actual usage
- Monitor resource consumption regularly
- Set warning thresholds to prevent service interruptions
- Consider offering tiered packages with different resource limits
</Tip>

## User Limitations

Control how many websites users can create and how frequently.

### Website Creation Limits

| Setting | Description | Default |
|---------|-------------|---------|
| **Maximum Websites per User** | Total number of websites a user can create | Varies |
| **Creation Delay** | Minimum time (in seconds) between website creations | 0 seconds |
| **Threshold Limit** | Maximum concurrent website creation requests | 1 |

**Maximum Websites per User**

Limit the total number of websites each user can create.

- **Purpose**: Prevent abuse and manage server resources
- **Recommended**: 5-10 for standard users, unlimited for premium

**Creation Delay (seconds)**

Minimum waiting time between creating websites.

- **Purpose**: Prevent rapid-fire website creation
- **Recommended**: 60-300 seconds (1-5 minutes)

**Threshold Limit**

Maximum number of websites a user can create simultaneously.

- **Purpose**: Prevent server overload
- **Recommended**: 1 (one at a time)

<Warning>
Setting limits too low may frustrate legitimate users. Balance security with user experience based on your platform's needs.
</Warning>

## Trial & Expiry Settings

Configure trial periods and website expiration behavior.

### Trial Period Configuration

**Trial Expiration Period (days)**

Number of days before a trial website expires.

- **Range**: 1 - 365 days
- **Recommended**: 7-30 days
- **Set to 0**: Disable trial expiration

**Delete After Expiration (days)**

Number of days to wait before permanently deleting expired websites.

- **Range**: 0 - 365 days
- **Recommended**: 7-14 days (gives users time to renew)
- **Set to 0**: Delete immediately upon expiration

### Standard Expiry Settings

**Default Expiry Period (days)**

Default number of days until a paid website expires (if not renewed).

- **Range**: 1 - 365 days
- **Common**: 30, 90, 180, or 365 days
- **Recommended**: 30 days for monthly subscriptions

<Info>
**How Expiration Works:**
1. Website reaches expiration date
2. Website is suspended/disabled
3. Grace period begins (Delete After Expiration)
4. If not renewed, website is permanently deleted
</Info>

<Tip>
**Expiration Best Practices:**
- Send email reminders before expiration
- Provide a grace period for renewals
- Clearly communicate expiration policies to users
- Consider auto-renewal options for paid plans
</Tip>

## Branding Assets

Upload custom images to personalize the website setup experience.

### Setup Image

**Website Setup Image**

Custom image displayed during website creation and setup process.

**Specifications:**
- **Recommended Size**: 1200 x 600 pixels
- **Format**: JPG, PNG, or GIF
- **Max File Size**: 2 MB
- **Purpose**: Branding and visual appeal during setup

**How to Upload:**

<Steps>
  <Step title="Click Upload Button">
    Select the image upload field in the Branding Assets section
  </Step>
  <Step title="Choose Image">
    Select your prepared image file from your computer
  </Step>
  <Step title="Preview">
    Review the uploaded image to ensure it displays correctly
  </Step>
  <Step title="Save Settings">
    Save your configuration to apply the branding
  </Step>
</Steps>

<Tip>
**Image Best Practices:**
- Use high-quality images that represent your brand
- Ensure images are optimized for web (compressed)
- Test images on different screen sizes
- Keep branding consistent across your platform
</Tip>

## Helpdesk Ticket Automation

Automate ticket management to improve efficiency and reduce manual work.

### Retention Settings

**Ticket Retention Period (days)**

Number of days to keep closed tickets before automatic deletion.

- **Range**: 1 - 365 days
- **Recommended**: 30-90 days
- **Purpose**: Maintain a clean database while preserving recent history

### Auto-Close Settings

**Auto-Close Awaiting Tickets (days)**

Automatically close tickets that have been awaiting customer response for this many days.

- **Range**: 1 - 365 days
- **Recommended**: 7-14 days
- **Purpose**: Prevent ticket backlog from abandoned conversations

**How Auto-Close Works:**

<Steps>
  <Step title="Ticket Awaits Response">
    Support team responds and ticket status changes to "Awaiting Customer"
  </Step>
  <Step title="Timer Starts">
    Auto-close countdown begins
  </Step>
  <Step title="Customer Doesn't Respond">
    If customer doesn't reply within the specified days
  </Step>
  <Step title="Ticket Auto-Closes">
    System automatically closes the ticket
  </Step>
  <Step title="Notification Sent">
    Customer receives notification about ticket closure
  </Step>
</Steps>

<Info>
**Automation Benefits:**
- Reduces manual ticket management
- Keeps helpdesk organized
- Improves response time metrics
- Frees up support team resources
</Info>

<Warning>
**Important Considerations:**
- Notify customers before auto-closing tickets
- Allow customers to reopen tickets if needed
- Don't set auto-close periods too short
- Monitor customer satisfaction after implementing automation
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Users Can't Create Websites" icon="circle-xmark">
    **Issue:** Users receive errors when trying to create websites
    
    **Common Causes:**
    - Reached maximum website limit
    - Creation delay not elapsed
    - Server resource constraints
    - Account restrictions
    
    **Solutions:**
    1. Check if user has reached their website limit
    2. Verify creation delay settings aren't too restrictive
    3. Review server resource availability
    4. Check for any account-level restrictions
    5. Review error logs for specific issues
  </Accordion>
  
  <Accordion title="Websites Expiring Too Quickly" icon="clock">
    **Issue:** Websites expire before users expect
    
    **Common Causes:**
    - Trial period set too short
    - Expiry settings misconfigured
    - No renewal reminders sent
    - Payment processing issues
    
    **Solutions:**
    1. Review and adjust trial period settings
    2. Verify expiry date calculations
    3. Set up automated renewal reminders
    4. Check payment gateway configuration
    5. Provide clear expiration information to users
  </Accordion>
  
  <Accordion title="Resource Limits Not Enforcing" icon="server">
    **Issue:** Users exceeding disk or bandwidth limits
    
    **Common Causes:**
    - Limits not properly configured
    - Monitoring not active
    - Cache issues
    - Legacy websites not updated
    
    **Solutions:**
    1. Verify resource limit settings are saved
    2. Check monitoring system is active
    3. Clear configuration cache
    4. Update existing websites with new limits
    5. Review server-level quotas
  </Accordion>
  
  <Accordion title="Helpdesk Tickets Not Auto-Closing" icon="ticket">
    **Issue:** Old tickets remain open indefinitely
    
    **Common Causes:**
    - Auto-close not enabled
    - Cron job not running
    - Ticket status incorrect
    - Configuration not saved
    
    **Solutions:**
    1. Verify auto-close settings are enabled
    2. Check cron jobs are running properly
    3. Review ticket status definitions
    4. Save configuration again
    5. Test with a sample ticket
  </Accordion>
  
  <Accordion title="Branding Image Not Displaying" icon="image">
    **Issue:** Custom setup image doesn't appear
    
    **Common Causes:**
    - Image not uploaded correctly
    - File size too large
    - Incorrect file format
    - Cache not cleared
    
    **Solutions:**
    1. Re-upload the image
    2. Compress image if too large
    3. Use supported formats (JPG, PNG, GIF)
    4. Clear browser and server cache
    5. Check file permissions
  </Accordion>
</AccordionGroup>
