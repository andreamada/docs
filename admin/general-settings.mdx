## Contact Button

Configure how customers can reach you through various communication channels.

### Contact Form & Helpdesk

**Enable Contact Form**

Show a contact form on your website for customer inquiries.

- **Enabled**: Contact form is visible to visitors
- **Disabled**: Contact form is hidden

**Enable Helpdesk Integration**

Integrate with your helpdesk system for ticket management.

- **Enabled**: Customer inquiries create support tickets
- **Disabled**: Inquiries are sent via email only

### Dynamic Contact Buttons

Add up to 20 customizable contact buttons to provide multiple ways for customers to reach you.

<Steps>
  <Step title="Choose Contact Method">
    Select from popular options like WhatsApp, Facebook Messenger, Telegram, or Email
  </Step>
  <Step title="Configure Button">
    Set the button title, link/URL, and background color
  </Step>
  <Step title="Arrange Order">
    Drag and drop to reorder buttons as needed
  </Step>
  <Step title="Save Configuration">
    Your contact buttons will appear on your website
  </Step>
</Steps>

#### Button Configuration Fields

| Field                | Description                         | Example                          |
| -------------------- | ----------------------------------- | -------------------------------- |
| **Button Title**     | Display text for the contact button | "Chat on WhatsApp", "Send Email" |
| **Contact Link**     | URL or link for the contact method  | `https://wa.me/1234567890`       |
| **Background Color** | Button background color (hex code)  | `#25D366` (WhatsApp green)       |

**Popular Contact Link Examples:**

```text WhatsApp
https://wa.me/1234567890
```


```text Facebook Messenger
https://m.me/username
```


```text Telegram
https://t.me/username
```


```text Email
mailto:contact@domain.com
```

</CodeGroup>

**Popular Brand Colors:**

- WhatsApp: `#25D366` (green)
- Facebook: `#1877F2` (blue)
- Telegram: `#0088CC` (light blue)
- Email: `#EA4335` (red)

<Tip>
  **Best Practices:**

  - Limit to 3-5 contact options to avoid overwhelming visitors
  - Use recognizable brand colors for each platform
  - Test all links before publishing
  - Keep button titles short and clear (e.g., "WhatsApp" instead of "Contact us on WhatsApp")
</Tip>

### Website Settings

Control how your website appears and behaves.

**Allow Search Engine Indexing** (Default: Enabled)

Let Google and other search engines index your website.

- **Enabled**: Your site appears in search results (recommended)
- **Disabled**: Your site is hidden from search engines

**Show Documentation Section** (Default: Enabled)

Display documentation on your homepage.

- **Enabled**: Documentation is visible to visitors
- **Disabled**: Documentation section is hidden

**Disable Landing Page** (Default: Disabled)

Skip the landing page and redirect directly to login.

- **Enabled**: Users go straight to login page
- **Disabled**: Users see the landing page first

### Tax Management

Configure tax settings for your platform.

**Enable Tax Management** (Default: Disabled)

Master toggle for tax functionality across the platform.

**Tax Label**

Name of the tax to display (e.g., "VAT", "GST", "Sales Tax")

**Tax Rate (%)**

Tax percentage to apply to transactions (e.g., enter "20" for 20%)

<Tip>
  **Tax Configuration Tips:**

  - Check your local tax regulations before enabling
  - Ensure your tax rate is accurate and up-to-date
  - Consider consulting with a tax professional for complex scenarios
  - Update tax settings when regulations change
</Tip>

### User Registration

**Allow User Registration** (Default: Disabled)

Enable public user account creation.

- **Enabled**: Visitors can create accounts
- **Disabled**: Only admins can create accounts

<Warning>
  Disabling user registration will prevent new customers from signing up. Only enable this if you want to control who can create accounts.
</Warning>

## License Settings

Manage and validate your software license to ensure continued access to updates and support.

### License Validation

<Steps>
  <Step title="Enter License Key">
    Input your license key in the provided field
  </Step>
  <Step title="Click Validate">
    Click the validation button to verify your license
  </Step>
  <Step title="Review Status">
    Check your license status, expiration date, and website count
  </Step>
  <Step title="Renew if Needed">
    If your license is expired or expiring soon, renew it to maintain access
  </Step>
</Steps>

### License Information

After validation, you'll see:

- ‚úÖ **License Status**: Active, Expired, or Invalid
- ‚úÖ **Expiration Date**: When your license expires
- ‚úÖ **Website Count**: Number of websites allowed under your license
- ‚úÖ **Domain Verification**: Confirms license is valid for your domain

<Warning>
  **Important:**

  - Keep your license key secure and confidential
  - Validate your license regularly to ensure it's active
  - Renew before expiration to avoid service interruptions
  - Contact support if you encounter validation errors
</Warning>

<Tip>
  **License Best Practices:**

  - Set a reminder to renew your license before it expires
  - Keep a backup copy of your license key in a secure location
  - Validate your license after any major system updates
  - Contact support immediately if your license shows as invalid
</Tip>

## Cron Automation

Configure automated tasks to run on a schedule, improving efficiency and reducing manual work.

### What is Cron Automation?

Cron automation allows you to schedule recurring tasks that run automatically at specified intervals. This is essential for:

<CardGroup cols={2}>
  <Card title="Database Cleanup" icon="database">
    Automatically remove old data and optimize database tables
  </Card>
  <Card title="Email Notifications" icon="envelope">
    Send scheduled emails, reminders, and notifications
  </Card>
  <Card title="Report Generation" icon="chart-line">
    Generate and distribute periodic reports automatically
  </Card>
  <Card title="Data Synchronization" icon="arrows-rotate">
    Sync data between systems and external services
  </Card>
</CardGroup>

### Setting Up Cron Jobs

<Steps>
  <Step title="Get Cron URL">
    Copy the cron endpoint URL from the Cron Automation tab
  </Step>
  <Step title="Access Server Cron">
    Log in to your server's control panel (cPanel, Plesk, etc.)
  </Step>
  <Step title="Create Cron Job">
    Add a new cron job with the URL you copied
  </Step>
  <Step title="Set Schedule">
    Configure how often the cron should run (e.g., every hour, daily)
  </Step>
  <Step title="Test Execution">
    Verify the cron job runs successfully
  </Step>
</Steps>

<Info>
  **Recommended Schedule:** Run cron jobs every 5-15 minutes for optimal performance. Adjust based on your specific needs and server resources.
</Info>

<Tip>
  **Cron Setup Tips:**

  - Start with a longer interval (e.g., hourly) and adjust as needed
  - Monitor server resources to ensure cron jobs don't overload your system
  - Set up logging to track cron job execution and errors
  - Test cron jobs in a staging environment first
  - Document your cron schedule for future reference
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="License Validation Fails" icon="key">
    **Issue:** License key shows as invalid or expired

    **Common Causes:**

    - License has expired
    - License key entered incorrectly
    - Domain doesn't match license
    - Network connectivity issues

    **Solutions:**

    1. Verify license key is copied correctly (no extra spaces)
    2. Check license expiration date
    3. Ensure domain matches the one registered for the license
    4. Try validating again after a few minutes
    5. Contact support if issue persists
  </Accordion>
  <Accordion title="Cron Jobs Not Running" icon="clock">
    **Issue:** Automated tasks are not executing

    **Common Causes:**

    - Cron job not configured on server
    - Incorrect cron URL
    - Server firewall blocking requests
    - Insufficient server resources

    **Solutions:**

    1. Verify cron job is set up in server control panel
    2. Check cron URL is correct and accessible
    3. Review server logs for errors
    4. Ensure server has enough resources
    5. Test cron URL manually in browser
  </Accordion>
  <Accordion title="Contact Buttons Not Appearing" icon="message">
    **Issue:** Contact buttons don't show on website

    **Common Causes:**

    - Buttons not saved properly
    - Cache not cleared
    - Theme compatibility issues
    - JavaScript errors

    **Solutions:**

    1. Save contact button configuration again
    2. Clear browser and server cache
    3. Check browser console for JavaScript errors
    4. Verify theme supports contact buttons
    5. Test on different browsers
  </Accordion>
  <Accordion title="Currency Changes Not Reflecting" icon="dollar-sign">
    **Issue:** Currency change doesn't update prices

    **Common Causes:**

    - Cache not cleared
    - Prices hard-coded in theme
    - Database not updated
    - Browser cache

    **Solutions:**

    1. Clear all caches (browser, server, application)
    2. Refresh pricing pages
    3. Update product prices manually if needed
    4. Check theme currency settings
    5. Verify database currency field is updated
  </Accordion>
  <Accordion title="Language Translations Missing" icon="language">
    **Issue:** Some text not translating correctly

    **Common Causes:**

    - Translation files incomplete
    - Language not fully enabled
    - Cache issues
    - Custom text not translated

    **Solutions:**

    1. Check translation files are complete
    2. Verify language is enabled in settings
    3. Clear language cache
    4. Add missing translations manually
    5. Re-upload language files if corrupted
  </Accordion>
</AccordionGroup>
