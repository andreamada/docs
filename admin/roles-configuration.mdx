## Overview

The User Roles & Permissions page allows you to create custom user roles and configure granular access permissions. Control what different users can see and do on your platform to maintain security and organize your team effectively.

## User Roles

Create and manage different user roles for your platform.

### Default Roles

Most platforms include these standard roles:

| Role | Typical Use | Access Level |
|------|-------------|--------------|
| **Administrator** | Platform owners, full access | Complete control |
| **Manager** | Team leads, supervisors | Most features |
| **Support** | Customer service team | Customer-facing features |
| **User** | Regular customers | Basic features only |
| **Guest** | Trial or limited users | Minimal access |

### Creating Custom Roles

<Steps>
  <Step title="Add New Role">
    Click "Add Role" to create a new user role
  </Step>
  <Step title="Name the Role">
    Choose a clear, descriptive name (e.g., "Sales Team", "Developer")
  </Step>
  <Step title="Set Permissions">
    Configure which features and areas this role can access
  </Step>
  <Step title="Review Access">
    Double-check permissions to ensure appropriate access
  </Step>
  <Step title="Save Role">
    Save the role to make it available for assignment
  </Step>
</Steps>

### Role Examples

**Sales Team Role:**
- View customer list
- Create new accounts
- Manage subscriptions
- View reports
- No access to: Settings, billing, user management

**Developer Role:**
- Access to API settings
- View logs and debugging tools
- Manage integrations
- No access to: Billing, customer data, pricing

**Accountant Role:**
- View all financial reports
- Manage invoices
- Process refunds
- Export financial data
- No access to: Technical settings, user passwords

<Tip>
**Role Design Tips:**
- Create roles based on job functions
- Follow principle of least privilege
- Regularly review and update permissions
- Document what each role can do
- Limit number of administrators
</Tip>

## Permission Categories

Permissions are organized into logical categories for easier management.

### Dashboard & Overview

**Dashboard Access**

Control access to the main dashboard and overview screens.

- **Full Access**: View all dashboard widgets and statistics
- **Limited Access**: View only relevant sections
- **No Access**: Dashboard hidden

### User Management

**User Permissions**

Control who can manage user accounts.

| Permission | Description |
|------------|-------------|
| **View Users** | See list of all users |
| **Create Users** | Add new user accounts |
| **Edit Users** | Modify existing user information |
| **Delete Users** | Remove user accounts |
| **Assign Roles** | Change user roles and permissions |

### Website Management

**Website Permissions**

Control access to website creation and management.

| Permission | Description |
|------------|-------------|
| **View Websites** | See list of all websites |
| **Create Websites** | Create new websites |
| **Edit Websites** | Modify website settings |
| **Delete Websites** | Remove websites |
| **Suspend Websites** | Temporarily disable websites |

### Domain Management

**Domain Permissions**

Control access to domain registration and management.

| Permission | Description |
|------------|-------------|

### Billing & Payments

**Financial Permissions**

| Permission | Description |
|------------|-------------|

| **Process Payments** | Accept and process payments |
| **Issue Refunds** | Process refund requests |
| **View Reports** | Access financial reports |
| **Manage Pricing** | Edit pricing and packages |

### Settings & Configuration

**System Permissions**

Control access to platform settings and configuration.

| Permission | Description |
|------------|-------------|
| **General Settings** | Modify platform settings |

### Support & Tickets

**Helpdesk Permissions**

| Permission | Description |
|------------|-------------|

## Permission Configuration

### Permission Levels

For each feature, you can typically set one of these levels:

**Full Access**
- Complete control over the feature
- Can view, create, edit, and delete
- Access to all related settings

**Read-Only Access**
- Can view information
- Cannot make changes
- Useful for reporting and oversight

**Limited Access**
- Can perform specific actions only
- Restricted to certain operations
- Custom limitations per role

**No Access**
- Feature completely hidden
- Cannot view or interact
- Most secure option

### Setting Permissions

<Steps>
  <Step title="Select Role">
    Choose the role you want to configure
  </Step>
  <Step title="Review Categories">
    Go through each permission category
  </Step>
  <Step title="Set Access Levels">
    Choose appropriate access level for each permission
  </Step>
  <Step title="Test Permissions">
    Log in as that role to verify access works correctly
  </Step>
  <Step title="Adjust as Needed">
    Fine-tune permissions based on testing
  </Step>
</Steps>

## Security Best Practices

<AccordionGroup>
  <Accordion title="Principle of Least Privilege" icon="shield">
    **Grant Minimum Necessary Access:**
    - Only give permissions required for job function
    - Start with minimal access and add as needed
    - Regularly review and remove unused permissions
    - Don't give admin access unless absolutely necessary
    
    **Regular Audits:**
    - Review user roles quarterly
    - Remove inactive users
    - Update permissions when roles change
    - Document permission changes
  </Accordion>
  
  <Accordion title="Role Separation" icon="users">
    **Separate Duties:**
    - Don't combine conflicting responsibilities
    - Separate financial and technical access
    - Require multiple approvals for sensitive actions
    - Implement checks and balances
    
    **Examples:**
    - Person who creates invoices shouldn't process refunds alone
    - Developers shouldn't have billing access
    - Support staff shouldn't modify pricing
  </Accordion>
  
  <Accordion title="Access Control" icon="lock">
    **Protect Sensitive Areas:**
    - Limit access to payment settings
    - Restrict API credential access
    - Control who can delete data
    - Monitor admin actions
    
    **Additional Security:**
    - Require two-factor authentication for admins
    - Log all permission changes
    - Set up alerts for suspicious activity
    - Regular security training for team
  </Accordion>
  
  <Accordion title="User Lifecycle Management" icon="user-clock">
    **Onboarding:**
    - Assign appropriate role immediately
    - Provide training on permissions
    - Document what they can access
    - Review after probation period
    
    **Offboarding:**
    - Immediately revoke access when employee leaves
    - Transfer ownership of their work
    - Review what they had access to
    - Change shared passwords they knew
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="User Can't Access Feature" icon="circle-xmark">
    **Issue:** User reports they cannot access a feature they need
    
    **Common Causes:**
    - Insufficient permissions for their role
    - Role not assigned correctly
    - Permission settings not saved
    - Cache not cleared
    
    **Solutions:**
    1. Verify user's assigned role
    2. Check role permissions for that feature
    3. Grant necessary permissions
    4. Save changes and clear cache
    5. Have user log out and back in
  </Accordion>
  
  <Accordion title="Permissions Not Applying" icon="shield-xmark">
    **Issue:** Permission changes don't take effect
    
    **Common Causes:**
    - Changes not saved
    - User session not refreshed
    - Cache issues
    - Browser cookies
    
    **Solutions:**
    1. Save permission changes again
    2. User must log out and log back in
    3. Clear application cache
    4. Clear browser cache and cookies
    5. Test in incognito/private mode
  </Accordion>
  
  <Accordion title="Too Many Administrators" icon="users">
    **Issue:** Security concern with multiple admin accounts
    
    **Common Causes:**
    - Default admin role assigned to everyone
    - No custom roles created
    - Lack of permission management
    
    **Solutions:**
    1. Audit all users with admin access
    2. Create appropriate custom roles
    3. Downgrade unnecessary admin accounts
    4. Implement role-based access control
    5. Regular permission reviews
  </Accordion>
  
  <Accordion title="User Has Wrong Permissions" icon="user-xmark">
    **Issue:** User can access things they shouldn't
    
    **Common Causes:**
    - Assigned to wrong role
    - Role has too many permissions
    - Multiple roles assigned
    - Permission inheritance issues
    
    **Solutions:**
    1. Check user's assigned role(s)
    2. Review role permissions
    3. Remove unnecessary roles
    4. Adjust role permissions
    5. Verify changes took effect
  </Accordion>
</AccordionGroup>

## Quick Reference

### Permission Checklist

**When Creating a New Role:**
- [ ] Choose clear, descriptive role name
- [ ] Define job function and responsibilities
- [ ] List required features and access
- [ ] Set minimum necessary permissions
- [ ] Test role with dummy account
- [ ] Document role purpose and permissions
- [ ] Review with security team
- [ ] Get approval before deployment

**When Assigning Roles:**
- [ ] Verify user identity
- [ ] Confirm job function
- [ ] Assign appropriate role
- [ ] Provide role documentation
- [ ] Train on permissions
- [ ] Set review date
- [ ] Log role assignment

**Regular Maintenance:**
- [ ] Quarterly permission audits
- [ ] Remove inactive users
- [ ] Update roles for job changes
- [ ] Review admin access
- [ ] Check for unused permissions
- [ ] Update documentation
- [ ] Security training refresher

### Common Permission Combinations

| Need | Permissions Required |
|------|---------------------|
| **View customer info** | View Users (read-only) |
| **Create accounts** | View Users + Create Users |
| **Manage support** | View/Reply/Close Tickets |
| **Process orders** | View Users + Create Invoices + Process Payments |
| **Technical support** | View/Edit Websites + Manage DNS + View Tickets |
| **Full admin** | All permissions |
